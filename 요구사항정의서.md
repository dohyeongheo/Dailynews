나는 Next.js, Tailwind CSS, Supabase를 사용하여 '매일 태국 및 한국 뉴스 요약 서비스'를 만들고 싶어.

아래 요구사항 정의서(RDD)를 바탕으로 프로젝트 구조를 잡고,
우선 1단계로 Supabase 테이블 생성 쿼리와, Google Gemini API를 연동하여
뉴스를 가져오는 백엔드 로직(Server Action)부터 작성해줘.

[요구사항 정의서]

1. 프로젝트 목표: 매일 아침 7시, Google AI API를 통해 최신 뉴스를 수집하고 DB에 저장 후 웹에 표출.
2. DB 스키마 (Supabase):

   - id (uuid, pk)
   - published_date (date)
   - source_country (text)
   - source_media (text)
   - title (text)
   - content (text)
   - content_translated (text, nullable)
   - category (text)
   - original_link (text)
   - created_at (timestamptz)

3. 뉴스 수집 로직 (Google Gemini API):

   - 중요: 최신 뉴스를 반영해야 하므로 Google Search Grounding 기능을 사용하거나 검색 도구를 활용해야 함.
   - 요청 프롬프트 구조: "{날짜} 의 태국 주요 뉴스(한국어 번역), 한국의 태국 관련 뉴스, 한국 주요 뉴스를 10개 이상 수집. JSON 포맷으로 출력."
   - JSON 응답을 파싱하여 DB에 저장하는 로직 필요.

4. 프론트엔드:

   - AWS 스타일의 심플하고 모던한 UI.
   - 헤더에 검색창 포함.
   - 메인 페이지에 3가지 카테고리(태국뉴스, 관련뉴스, 한국뉴스) 섹션 구분.

이 내용을 바탕으로 먼저 `supabase/schema.sql` 파일과 `lib/news-fetcher.ts` (API 호출 로직) 코드를 작성해줘.
