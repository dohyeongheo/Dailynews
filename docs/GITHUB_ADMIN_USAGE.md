# GitHub 관리 기능 사용 가이드

## 개요

관리자 페이지에서 GitHub 워크플로우 추적 및 프로젝트 관리(이슈, PR, 릴리즈) 기능을 사용하는 방법을 설명합니다.

## 접근 방법

1. 관리자 페이지 접속: `/admin`
2. "GitHub 관리" 탭 클릭
3. 하위 탭에서 원하는 기능 선택

## 워크플로우 관리

### 기능

- 워크플로우 목록 조회
- 실행 기록 조회 및 필터링
- 실행 상세 정보 확인
- 실패한 워크플로우 재실행

### 사용 방법

1. **워크플로우 탭** 클릭
2. **필터 설정**:
   - 워크플로우 선택: 특정 워크플로우만 보기
   - 상태 필터: 대기 중, 실행 중, 완료
   - 결과 필터: 성공, 실패, 취소됨, 건너뜀
3. **실행 기록 확인**:
   - 실행 번호, 워크플로우 이름, 브랜치, 이벤트, 상태 확인
   - "보기" 버튼으로 GitHub에서 상세 정보 확인
4. **재실행**:
   - 실패한 워크플로우의 "재실행" 버튼 클릭

### 예시

- "Fetch News Daily" 워크플로우의 최근 실패한 실행 확인
- 실패한 실행을 재실행하여 문제 해결 확인

## 이슈 관리

### 기능

- 이슈 목록 조회 (전체/열림/닫힘)
- 이슈 생성
- 이슈 상세 정보 확인

### 사용 방법

1. **이슈 탭** 클릭
2. **이슈 목록 확인**:
   - 상태 필터로 열림/닫힘 이슈 필터링
   - 이슈 제목, 상태, 라벨, 작성자 정보 확인
3. **이슈 생성**:
   - "이슈 생성" 버튼 클릭
   - 제목, 내용, 라벨 입력
   - "생성" 버튼 클릭

### 예시

- 워크플로우 실패에 대한 이슈 생성
- 버그 리포트 이슈 생성

## Pull Request 관리

### 기능

- PR 목록 조회 (전체/열림/닫힘)
- PR 생성
- PR 머지

### 사용 방법

1. **Pull Requests 탭** 클릭
2. **PR 목록 확인**:
   - 상태 필터로 열림/닫힘 PR 필터링
   - PR 제목, 상태, 브랜치 정보 확인
3. **PR 생성**:
   - "PR 생성" 버튼 클릭
   - 제목, Head 브랜치, Base 브랜치, 내용 입력
   - "생성" 버튼 클릭
4. **PR 머지**:
   - 머지 가능한 PR의 "머지" 버튼 클릭

### 예시

- 기능 브랜치에서 main으로 PR 생성
- 코드 리뷰 후 PR 머지

## 릴리즈 관리

### 기능

- 릴리즈 목록 조회
- 릴리즈 생성

### 사용 방법

1. **릴리즈 탭** 클릭
2. **릴리즈 목록 확인**:
   - 릴리즈 이름, 태그, 작성자, 발행일 확인
3. **릴리즈 생성**:
   - "릴리즈 생성" 버튼 클릭
   - 태그 이름, 릴리즈 이름, 릴리즈 노트 입력
   - 초안/프리릴리즈 옵션 선택
   - "생성" 버튼 클릭

### 예시

- v1.0.0 릴리즈 생성
- 프리릴리즈로 베타 버전 발행

## AI 에이전트 활용

### 자동 에러 분석

관리자 페이지에서 직접 사용할 수는 없지만, AI 에이전트(Cursor AI)가 다음 기능을 활용할 수 있습니다:

1. **워크플로우 실패 자동 감지**: "오늘 실패한 워크플로우를 찾아서 원인을 분석해줘"
2. **에러 패턴 분석**: "최근 일주일간 워크플로우 실패 트렌드를 분석해줘"
3. **자동 이슈 생성**: "실패한 워크플로우에 대해 이슈를 자동으로 생성해줘"
4. **코드 수정 제안**: "이 워크플로우 실패를 해결하기 위한 코드 수정을 제안해줘"

자세한 내용은 [AI 에이전트 에러 트래킹 분석 문서](./AI_AGENT_ERROR_TRACKING_ANALYSIS.md)를 참조하세요.

## 환경 변수 설정

GitHub 관리 기능을 사용하려면 다음 환경 변수가 설정되어 있어야 합니다:

- `GITHUB_TOKEN` 또는 `GITHUB_PERSONAL_ACCESS_TOKEN`: GitHub Personal Access Token
- `GITHUB_OWNER` (선택사항): 리포지토리 소유자 (기본값: dohyeongheo)
- `GITHUB_REPO` (선택사항): 리포지토리 이름 (기본값: Dailynews)

### GitHub Token 생성 방법

1. GitHub Settings > Developer settings > Personal access tokens > Tokens (classic)
2. "Generate new token" 클릭
3. 필요한 권한 선택:
   - `repo` (전체 저장소 접근)
   - `actions:read` (워크플로우 실행 기록 조회)
   - `workflow:read` (워크플로우 읽기)
4. 토큰 생성 후 환경 변수에 설정

## 문제 해결

### "GITHUB_TOKEN이 설정되지 않았습니다" 오류

- 환경 변수에 `GITHUB_TOKEN` 또는 `GITHUB_PERSONAL_ACCESS_TOKEN`이 설정되어 있는지 확인
- 서버 재시작 필요할 수 있음

### "Rate Limit이 초과되었습니다" 오류

- GitHub API Rate Limit에 도달
- 잠시 후 다시 시도

### 데이터가 표시되지 않음

- 관리자 권한이 있는지 확인
- GitHub Token 권한이 올바른지 확인
- 네트워크 연결 확인
